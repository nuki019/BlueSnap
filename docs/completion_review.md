## 2. 复赛叙事

大叙事：蓝心快搭面向“有创造力但不一定会开发的人”：学生、社团组织者、小微商家、内容创作者、教师、自由职业者和生活效率爱好者。  
核心问题：很多真实需求不是“做一个大 App”，而是“现在就需要一个刚好适合我的小工具”。现有应用太重、太泛，低代码仍有学习门槛，纯代码生成又需要开发环境。  
一句话价值：把“我有一个想法”变成“手机里立即能跑、能分享、能离线保存的轻应用”。  
黄金样例：18-24 岁高校社团负责人/班委的活动筹备工具，用来证明这套范式真实可用。

不要在复赛材料中写成已实现：
- 端侧蓝星 3B 本地推理
- 系统 API 桥接调用相机、短信、健康数据
- 一键生成 APK 或应用市场
- 自动恶意代码扫描平台

可以明确展示的差异化：
- 手机端原生入口，不需要电脑和开发环境
- 需求对话 -> 方案确认 -> HTML 生成 -> WebView 运行的闭环
- 功能清单可开关，用户在生成前能控制范围
- WebView 沙箱运行，默认阻断外部跳转和外部资源
- Mock 兜底保障演示稳定，真实模型作为可配置能力
- HTML 可分享、可离线导出，适合轻量工具流转和留存

## 3. 黄金演示链路

演示输入：
> 给迎新活动做一个报名、分工和预算管理工具，要能记录参与同学、负责人、支出金额和筹备进度。

视频脚本（建议 2分30秒）：
1. 0:00-0:15 展示大叙事：创造力不应卡在“不会开发”上；社团活动是具体样例。
2. 0:15-0:35 打开蓝心快搭，点击“活动筹备”或输入上方需求。
3. 0:35-1:05 展示 AI 流式理解和方案确认，强调功能开关。
4. 1:05-1:35 确认生成并进入 WebView 预览。
5. 1:35-2:10 在活动筹备助手中添加报名、任务、预算，展示数据变化。
6. 2:10-2:25 点击导出/分享 HTML，说明可离线保存并传给同学。
7. 2:25-2:40 收尾展示差异化：手机端闭环、即开即用、安全沙箱。

真实截图清单：
- 首页：活动筹备模板位于首位
- 对话页：需求被理解并流式回复
- 方案页：报名名单、任务分工、预算记录、进度打卡
- 预览页：活动筹备助手运行中
- 分享入口：顶部分享 HTML 按钮

## 4. PPT 结构

要求：禁用流程图；每页用真实截图、数据卡或对比表表达。

1. 封面：蓝心快搭 + 宣传语 + 手机真实截图
2. 用户洞察：有创造力的人 + 具体样例人群，3条访谈摘录，1张数据卡
3. 核心问题：长尾小工具需求高频但碎片化
4. 方案展示：真实首页截图 + 一句话输入
5. 主流程两步：描述需求、确认生成并运行
6. 差异化：手机端闭环、WebView 沙箱、离线导出/分享 HTML、Mock 兜底
7. 完成度：APK 可运行、核心模块列表、演示场景
8. 数据验证：生成测试、访谈反馈、愿意使用/付费意愿
9. 安全设计：key 不入库、默认 demo-safe、外链拦截、备份关闭
10. 结尾：下一阶段只写端侧模型/系统 API 为规划，不写成已完成
