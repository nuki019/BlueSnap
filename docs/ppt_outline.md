# 蓝心快搭复赛 PPT 完整提纲

## 0. 制作原则

- 主标题：蓝心快搭：校园效率轻工具生成器
- 核心口径：一句话，把校园里的临时需求变成手机里可运行、可保存、可分享的轻工具。
- 宣传语：把创造的自由还给用户。
- 页数建议：12-14 页；如果时间紧，压缩到 10-12 页。
- 表达要求：禁用流程图；每页优先使用真实 App 截图、数据卡、对比表、演示截图。
- 不要写成已实现：端侧大模型、系统 API 桥接、APK 生成器、云端发布平台。

## 1. 封面

标题：蓝心快搭 BlueSnap  
副标题：校园效率轻工具生成器  
宣传语：把创造的自由还给用户

画面建议：
- 手机首页真实截图：蓝色品牌区、四张场景卡和浮层底部导航。
- WebView 预览页截图。
- 分享成果页截图。

讲述重点：
- 先让评委知道这不是概念图，而是已经能在手机里跑的作品。
- 第一页就露出“首页入口、运行效果、分享留存”三个完成度信号。

## 2. 用户是谁

标题：面向 18-24 岁大学生和准职场青年

核心内容：
- 大学生：社团活动、课程任务、小组作业、考试复习、生活预算。
- 准职场青年：求职投递、实习面试、项目协作、习惯打卡。
- 他们不一定会开发，但经常需要临时、小型、个性化的工具。

画面建议：
- 两张用户画像卡片：大学生、准职场青年。
- 每张卡片包含 3 个典型需求，不放虚构头像也可以。

## 3. 用户洞察

标题：校园里的工具需求很多，但现成工具经常“不刚好”

访谈摘录建议：
- “只是想做一次活动报名，不想下载复杂 App。”
- “小组作业分工每次都靠群消息，很容易乱。”
- “求职投递表自己做很麻烦，但专门软件又太重。”

数据卡建议：
- 访谈人数：8-12 人。
- 生成测试样例：10-20 条。
- 最常见需求 Top 3：活动筹备、课程协作、求职投递。
- 愿意尝试 AI 生成轻工具比例：待补实测。

## 4. 核心问题

标题：他们不是想开发 App，而是现在就需要一个小工具

对比表：

| 方案 | 主要问题 |
|---|---|
| 下载现成 App | 功能太重，不贴合一次性场景 |
| Excel / 群消息 | 信息分散，协作混乱 |
| 低代码平台 | 有学习门槛，需要配置 |
| AI 代码生成 | 需要电脑、环境和调试能力 |

结论句：
- 校园长尾需求需要的是“即说即用、刚好够用、能分享留存”的轻工具。

## 5. 产品定义

标题：蓝心快搭：一句话生成校园轻工具

核心内容：
- 用户用自然语言描述需求。
- AI 生成可确认的功能方案。
- AI 生成单页 HTML 轻应用。
- Android WebView 沙箱中即时运行。
- 生成结果可导出、可分享。

画面建议：
- 首页真实截图。
- 圈出四个校园场景卡和自然语言输入框。

## 6. 首页入口设计

标题：从校园场景出发，而不是让用户想功能

四个一级入口：
- 活动筹备：报名、分工、预算、进度。
- 课程协作：作业、DDL、小组任务。
- 求职实习：投递、面试、反馈。
- 生活管理：预算、习惯、日常记录。

讲述重点：
- 用户不需要理解“生成应用”的技术概念，只要从自己的场景开始。
- 入口页已经重构为复赛口径一致的校园场景卡。

画面建议：
- 首页完整截图，蓝色标题区、四张白色场景卡、底部浮层导航必须清晰可见。

## 7. 黄金演示场景

标题：活动筹备助手：把一次校园活动管清楚

演示输入：
> 帮我做一个校园活动筹备工具，要能记录报名、任务分工、预算和进度。

生成能力：
- 报名名单。
- 任务分工。
- 预算记录。
- 进度打卡。

画面建议：
- 对话页截图。
- 方案确认页截图。

讲述重点：
- 活动筹备是黄金样例，不代表产品只做社团活动。
- 这个样例证明“描述需求 -> 方案确认 -> 运行工具 -> 分享留存”的闭环可用。

## 8. 用户可控生成

标题：AI 先给方案，用户确认后再生成

核心内容：
- 功能清单可查看。
- 功能开关可调整。
- 布局风格可选择。
- 确认后再生成应用。

画面建议：
- 方案确认页真实截图。
- 用标注框突出功能开关。

讲述重点：
- AI 不是直接替用户决定，而是先把需求变成可确认的方案。
- 这一步降低模型自由发挥带来的不稳定。

## 9. 最终运行效果

标题：生成结果不是代码，而是手机里能操作的工具

核心内容：
- 活动筹备助手在 WebView 中真实运行。
- 可以添加参与同学。
- 可以新增分工任务。
- 可以记录预算支出。
- 可以标记筹备进度。

画面建议：
- WebView 预览页截图。
- 截图里必须有已添加的数据，不要用空白状态。

## 10. 分享与留存

标题：从“生成出来”到“真的能用、能传”

核心内容：
- 新增“分享成果”页。
- 新增“历史记录”页，可再次打开、删除、分享生成过的轻工具。
- 展示应用名称和用途摘要。
- 导出 HTML 到本地。
- 调用系统分享 HTML。
- 不依赖公网发布，不需要云托管。

画面建议：
- 历史记录页截图，至少保留一个生成结果卡片。
- 分享成果页截图。
- 标注“导出 HTML”和“系统分享 HTML”两个按钮。

讲述重点：
- 轻工具的价值不只是生成，还要能留存和流转。
- HTML 文件适合发给同学，也适合离线保存。

## 11. 差异化价值

标题：不做云 IDE，做手机端轻工具闭环

对比表：

| 维度 | 云端 AI 编程工具 | 蓝心快搭 |
|---|---|---|
| 使用入口 | 浏览器 / IDE | Android 手机 |
| 目标 | 完整 Web / App 项目 | 校园轻工具 |
| 用户要求 | 需要理解工程概念 | 只需描述需求 |
| 运行方式 | 云端部署或代码环境 | WebView 即时运行 |
| 分享方式 | 链接 / 部署 | HTML 导出 / 系统分享 |
| 演示稳定性 | 依赖网络和模型 | Mock 兜底 + demo-safe |

讲述重点：
- 不说“比 Replit/Bolt 更强”。
- 说“我们选择了更小、更确定、更适合手机现场演示的切口”。

## 12. 完成度与安全设计

标题：复赛阶段已形成可运行、可演示、可提交版本

完成度：
- APK 可构建运行。
- 首页场景入口完成。
- 浮层底部导航和历史记录完成。
- AI 调用与 Mock 兜底完成。
- 活动筹备黄金模板完成。
- 快捷思维导图固定模板完成。
- WebView 沙箱完成。
- HTML 导出/分享完成。
- 分享成果页完成。

安全设计：
- 真实 API Key 不入库。
- 默认 demo-safe 模式。
- WebView 阻断外部跳转和外部资源。
- 禁止 file/content 访问。
- APK 与源码扫描无真实密钥。

画面建议：
- 完成度表。
- 安全检查结果卡片。
