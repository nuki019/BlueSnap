# 蓝心快搭 -- 基于蓝心大模型的AI驱动移动端应用生成平台

## 第三届(2026)"中国高校计算机大赛-AIGC创新赛"应用赛道 初赛作品策划书

---

## 一、项目概述

### 1.1 作品信息

| 项目 | 内容 |
|------|------|
| 作品名称 | 蓝心快搭 |
| 赛道分类 | 应用赛道 -- 系统工具 |
| 产品定位 | 基于vivo蓝心大模型的AI驱动零代码移动端应用生成平台 |
| 核心标语 | 将创作的自由还给每个人 |
| 技术路线 | 端侧优先、端云协同 |
| 适配终端 | vivo手机（OriginOS） |

### 1.2 一句话定义

> 用户用自然语言描述需求，蓝心大模型自动生成可直接运行的移动应用。

### 1.3 核心价值主张

将应用开发的门槛从"会写代码"降至"会说需求"。每一位vivo用户都能成为应用的创造者——无需编程知识，无需环境配置，只需说出想法，即可获得可运行的专属工具。

## 二、设计理念

### 2.1 核心理念：对话即开发 (Dialogue as Development)

**"将创作的自由还给每个人"**

蓝心智造不是"AI替你做应用"，而是"AI帮你说的话变成应用"。用户始终是创作者，AI的角色是翻译官与执行者：

- **用户主导**：AI不替用户做决策，只负责将自然语言准确转化为可运行的应用
- **全程可控**：需求描述 -> 方案确认 -> 应用生成 -> 迭代修改，每个环节用户均可参与决策
- **所见即所得**：生成的应用在工具内即时预览运行，无需外部环境

### 2.2 设计哲学

| 原则 | 内涵 |
|------|------|
| 对话即开发 | 将软件工程流程转化为自然语言对话，降低认知负荷 |
| 轻量化优先 | 单HTML文件形态，几十KB即可承载完整应用 |
| 渐进式能力 | 从轻量工具到完整APP，用户可随需求深入解锁高阶能力 |
| 生态深度融合 | 充分发挥蓝心大模型端侧推理与AI OS系统能力，实现原生级体验 |

### 2.3 蓝心大模型能力融合策略

蓝心大模型矩阵包含十亿、百亿、千亿三个参数量级的五款大模型，蓝心智造在产品各环节深度调用：

| 产品环节 | 调用的蓝心能力 | 说明 |
|----------|--------------|------|
| 需求理解 | 多模态意图识别 | 解析用户描述的功能需求、使用场景、目标画像 |
| 方案设计 | 逻辑推理与规划 | 生成功能架构、交互流程、界面布局方案 |
| 代码生成 | 代码生成专项能力 | 输出完整可运行的HTML/CSS/JavaScript代码 |
| 迭代优化 | 上下文理解与补丁推理 | 基于现有代码进行局部修改，而非全量重写 |
| 端侧运行 | 蓝心3B端侧多模态模型 | 离线场景下的轻量推理，保障隐私 |

## 三、需求分析与痛点

### 3.1 核心痛点

**痛点一：长尾需求真空**
用户在生活中常有极度碎片化、个性化的功能需求（如：一个专门记录某种植物浇水频率的工具、一个特定格式的课堂签到表）。这类需求市场空间小，在应用商店几乎不可能找到完全匹配的产品。

**痛点二：开发门槛极高**
传统移动开发涉及环境配置、语言学习（Java/Kotlin/Swift）、UI设计、编译打包等，对普通用户而言是不可能完成的任务。即便是低代码平台，仍需理解组件、数据绑定等概念。

**痛点三：应用过载与臃肿**
为了一个微小功能去下载数十甚至数百兆的App，不仅占用存储空间，更增加了日常使用的复杂性。用户需要的是"用完即走"的轻量工具，而非又一个重量级应用。

### 3.2 目标用户画像

| 用户群 | 特征 | 典型场景 |
|--------|------|----------|
| 大学生 | 学习效率工具需求旺盛，时间碎片化 | 番茄钟、单词卡、课程表、小组分工表 |
| 职场人士 | 个性化办公需求多样，IT支持有限 | 会议纪要整理、客户名片管理、快速审批流 |
| 小微商家 | 有数字化需求但无技术预算 | 简易记账、库存盘点、客户预约表 |
| 创意爱好者 | 有想法但缺乏实现能力 | 个人作品集、创意小游戏、生活记录工具 |

### 3.3 场景覆盖矩阵

覆盖赛事要求的五大实际使用场景：

| 场景 | 可生成的应用类型 | 结合的蓝心能力 |
|------|----------------|--------------|
| 移动办公 | 会议纪要整理、任务看板、快速审批应用 | 文本总结、语音识别、文档矫正 |
| 学习教育 | 单词记忆卡、课程表管理、知识问答卡片 | 文案创作、多模态理解、智能问答 |
| 出行生活 | 行程规划助手、消费记账本、健康打卡工具 | 意图识别、个性化推荐、端侧推理 |
| 影像创作 | 图片标注工具、简易修图助手、拍照记事 | 视觉多模态理解、AI消除、图片打标 |
| 系统工具 | 番茄钟、习惯追踪器、快捷设置面板 | 系统框架组件、端侧本地运行、原子化能力组合 |

```
│   └── 系统API桥接层

│   ├── 历史记录管理

└── 6. 迭代优化模块
    ├── 对话式修改指令

    ├── 多版本回溯
    └── 摇一摇唤起反馈面板
```

### 4.2 核心业务流程

```
用户输入需求（自然语言/语音/模板选择）
        │
        ▼
蓝心大模型意图解析与需求结构化
        │
        ├── 信息不足？ ──> 智能追问补充关键信息
        │
        ▼
生成设计方案（功能清单 + 布局方案 + 交互流程）
        │
        ├── 用户不满意？ ──> 调整需求重新生成
        │
        ▼
用户确认方案
        │
        ▼
一键生成单HTML应用代码
        │
        ▼
WebView沙箱内实时预览运行
        │
        ├── 需要修改？ ──> 对话式迭代优化 ──> 增量代码更新
        │
        ▼
满意 ──> 导出/分享/固定到桌面
```

### 4.3 各功能模块详细设计

#### 模块一：自然语言需求输入

- **多轮对话**：支持用户逐步补充、修改需求，AI持续追踪上下文
- **场景模板**：预置番茄钟、待办清单、习惯打卡、记账本等高频场景，一键快速启动
- **语音输入**：调用蓝心大模型语音识别能力，支持口述需求
- **智能追问**：当需求描述模糊时，AI主动提问澄清（如"你需要这个计时器支持暂停吗？"）
- **需求结构化**：后台将自然语言自动转化为结构化的需求描述（功能点列表、交互约束、视觉偏好）

#### 模块二：智能方案生成与确认

- **功能清单**：自动生成带优先级排序的功能点列表，用户可通过开关增删功能项
- **布局方案**：提供2-3种界面布局缩略图方案，用户点选切换
- **交互流程**：以简洁流程图展示核心交互路径
- **方案回溯**：支持回到任意步骤重新调整，不丢失已确认的内容

#### 模块三：一键HTML应用生成

- **代码生成**：基于确认方案，调用蓝心大模型生成完整单HTML文件（内含CSS样式 + JavaScript交互逻辑）
- **流式输出**：生成过程可视化，实时展示代码构建进度，增加科技感与信任感
- **质量校验**：内置HTML/CSS/JS语法检查，自动修复常见错误
- **移动适配**：自动生成响应式布局，适配不同vivo机型的屏幕尺寸

#### 模块四：内置实时预览运行

- **安全沙箱**：所有生成应用在隔离的WebView容器中运行，无法直接访问系统敏感资源
- **沉浸预览**：全屏展示生成的应用，顶部保留可隐藏的编辑/反馈悬浮球
- **热更新**：修改需求后即时重新生成并刷新预览，无需手动操作
- **系统API桥接**：通过统一封装的接口层，允许生成的应用安全调用摄像头、日历、通知等系统能力

#### 模块五：应用管理与分享

- **卡片流管理**：以卡片形式展示用户生成的所有作品，支持列表/网格切换
- **一键导出**：导出为独立HTML文件，离线可用
- **分享能力**：通过vivo系统分享功能将应用传递给其他用户，对方可直接打开使用
- **桌面固定**：支持将常用工具添加到桌面快捷方式

#### 模块六：迭代优化与反馈

- **对话式修改**：用户通过自然语言描述修改需求（如"把背景改成淡蓝色"、"按钮调大一点"），AI基于现有代码进行增量更新
- **版本管理**：记录每次修改的历史版本，支持回溯到任意版本
- **摇一摇反馈**：在预览页面摇一摇唤起反馈面板，快速描述不满意的地方

### 4.4 远期展望功能

初赛阶段聚焦单HTML文件生成与内置WebView预览的轻量化版本，以下功能作为远期演进方向：

## 五、技术架构

### 5.1 整体架构设计

采用"端侧优先、端云协同"策略，与vivo蓝心智能战略的技术路线高度一致：

```
┌─────────────────────────────────────────────────────┐
│                    交互层 (UI)                        │
│  自然语言输入 │ 方案确认卡片 │ 预览窗口 │ 编辑面板    │
├─────────────────────────────────────────────────────┤
│                 智能引擎层 (AI Core)                  │
│  ┌───────────┐ ┌───────────┐ ┌───────────────────┐  │
│  │ 需求解析   │ │ 方案生成   │ │ 代码生成           │  │
│  │ 模块       │ │ 模块       │ │ 模块               │  │
│  └───────────┘ └───────────┘ └───────────────────┘  │
├─────────────────────────────────────────────────────┤
│              系统能力层 (vivo AI OS)                  │
│  ┌──────────────────────────────────────────────┐   │
│  │ 端侧AI服务 │ 意图框架2.0 │ 系统级API │ MCP/A2A │   │
│  └──────────────────────────────────────────────┘   │
├─────────────────────────────────────────────────────┤
│               运行环境层 (Runtime)                    │
│  WebView渲染引擎 │ 本地存储沙箱 │ APK打包工具链       │
└─────────────────────────────────────────────────────┘
```

### 5.2 分层能力利用

| 层级 | vivo开放能力 | 智造APP调用方式 |
|------|-------------|----------------|
| 应用级 | 文本创作、文本总结、小V写作等开箱即用API | 嵌入生成应用的功能模块（自动摘要、智能填表） |
| 原子级 | 文本处理、多模态理解、视觉处理、语音处理等原子API | 灵活组合实现生成应用中的各类AI功能 |
| 部署级 | VCAP计算加速平台 | 为生成的复杂应用提供算法端侧部署加速 |

### 5.3 端云协同调度策略

| 任务类型 | 执行位置 | 原因 |
|----------|---------|------|
| 需求意图解析（简单） | 端侧（蓝心3B） | 低延迟、保隐私、离线可用 |
| 方案设计生成 | 端侧/云端（动态决策） | 根据复杂度智能选择 |
| 复杂代码生成 | 云端（百亿/千亿级模型） | 需要更强推理能力 |
| 迭代增量修改 | 端侧/云端（动态决策） | 简单修改端侧处理，复杂重构云端处理 |
| 代码质量校验 | 端侧 | 规则引擎，无需大模型 |

### 5.4 安全机制

- **沙箱隔离**：所有生成的应用在独立WebView容器中运行，与主应用进程隔离
- **权限控制**：系统API调用需经过统一网关，用户显式授权后才可执行
- **代码扫描**：自动检测危险代码模式（XSS、恶意重定向、未授权数据访问等），命中则拦截
- **隐私保护**：端侧推理数据不上传云端，敏感需求仅在本地处理

### 5.5 初赛阶段技术方案

初赛聚焦轻量化版本，技术栈选择如下：

| 组件 | 技术选型 | 说明 |
|------|---------|------|
| 移动端框架 | Kotlin + Jetpack Compose | vivo原生Android开发 |
| UI渲染 | WebView (Android内置) | 生成应用的运行沙箱 |
| 大模型接入 | 蓝心大模型API (云端) | 需求解析、方案生成、代码生成 |
| 端侧推理 | 蓝心3B端侧模型 SDK | 简单意图解析、离线场景 |
| 本地存储 | Room + SharedPreferences | 应用历史记录、用户偏好 |
| 代码校验 | 自研规则引擎 | HTML/CSS/JS语法检查与自动修复 |

## 七、核心创新点

### 7.1 形态创新："对话即开发"的全新应用创建范式

打破"先下载、后使用"的传统应用模式，开创"先描述、后生成、即使用"的新范式。不同于低代码平台的"拖拉拽"模式，用户无需理解任何技术概念，仅用自然语言交互即可完成从创意到运行的全流程。将应用开发从工程问题转化为对话问题。

### 7.2 技术创新：端侧AI与云端大模型协同的智能生成架构

行业多数AI应用生成工具依赖纯云端大模型，存在延迟高、断网不可用、隐私风险等问题。蓝心智造创新性地采用"端侧优先、端云协同"架构：

- **端侧（蓝心3B）**：处理需求解析与方案生成等轻量任务，保障离线可用与用户隐私
- **云端（百亿/千亿级模型）**：应对高度复杂的代码生成场景
- **智能分层调度**：根据任务复杂度自动选择执行层级，实现性能、隐私与效果的最佳平衡

### 7.3 生态创新：深度融入vivo AI OS的系统级应用生成

不同于同类工具仅生成纯Web应用，蓝心智造通过系统API桥接层，使生成的应用能够安全调用设备硬件与系统能力：

- 生成的记账应用可读取短信验证码自动填单
- 生成的健康助手可读写系统健康数据
- 生成的拍照工具可调用摄像头并利用AI影像处理

这种系统级的深度融合，使生成的应用具备了真正的实用价值，而非仅仅是网页玩具。

### 7.4 体验创新：生成+运行的完整移动端闭环

不同于多数同类工具"生成代码 -> 导出 -> 用户自己想办法运行"的断裂流程，蓝心智造提供 **生成 -> 预览 -> 运行 -> 迭代 -> 分享** 的完整闭环。用户始终在同一个移动端环境中完成所有操作，极大降低使用阻力。单HTML文件形态下应用体积仅几十KB，秒级生成、即时运行。

### 7.5 框架创新：蓝心个人智能框架赋能的个性化生成

通过接入蓝心个人智能框架：

- **全域感知**：理解用户使用场景、偏好、设备状态，生成更贴合个人需求的应用
- **个人记忆**：沉淀用户的应用使用习惯与偏好，后续生成越来越精准
- **意图框架2.0**：兼容MCP协议与A2A智能体协议，实现更精准的需求理解与Agent间协同

## 八、市场前景评估

### 8.1 行业趋势

**AI应用生成赛道全面爆发**：2025-2026年，阿里巴巴Meoo、蚂蚁灵光、字节扣子2.5、美团NoCode等产品密集涌现，印证了巨大的市场空间。移动端AI应用生成工具的每一次生成、每一轮修改都对应着算力消耗，是新一轮Token消耗战的核心入口。

**低代码/无代码市场高速增长**：据Gartner预测，到2025年70%的新应用将通过低代码/无代码工具开发。中国低代码市场规模已突破131亿元，年复合增长率超20%。全球无代码AI平台市场从2024年的55.5亿美元增长至2025年的70.6亿美元（CAGR 27.3%）。

**移动端原生体验存在缺口**：目前主流AI开发工具（GitHub Spark、阿里Meoo等）多聚焦Web端，移动端原生体验存在明显缺口。vivo依托庞大的移动端设备基础和系统深度整合能力，具备打造差异化产品的绝佳窗口。

**端侧AI成为手机标配**：2026年旗舰手机NPU算力同比提升3倍，大模型经轻量化优化体积缩小70%，运行功耗降低50%。vivo计划未来3-5年让超3亿设备具备本地AI能力。
